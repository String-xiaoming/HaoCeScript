# Haoce Auto Reader

**贵理工专用**

GPT-5.4写的

测试机型是 红米 note 10 pro(root),非全面屏状态下

目前就剩些小bug待出现了,大体可以自动刷了,就是为了模拟真实阅读情况刷的会有点慢.

**js文件夹里的.js,可以在安卓无障碍运行,这样就不用adb连接来刷了**

这是一个基于 ADB 的好策自动阅读脚本。

它会做这些事：
- 启动 `app.haoce.com`
- 优先进入当前阅读中的书
- 在正文里自动上滑
- 通过截图差分判断“是否已经到底”
- 到底后自动左右翻页进入下一节
- 如果一本到最后一章最后一页，翻页失败后会回首页确认这本书是否已经 `100%`
- 确认读完后，会去首页左上角的 `书香理工` 书单页找下一本未读完的书继续读
- 如果脚本一启动就发现 `最近阅读` 第一栏已经是 `100%`，也会直接去 `书香理工` 书单页找下一本
- 如果应用上次就停在 `书香理工` 书单页，脚本也能直接从当前书单页继续选分类和选书

## 先看这里

你最常改的地方只有这几个。

### 1. 改 15 秒停留

看 [config.json](/E:/haoce/config.json) 里的这段：

```json
"scroll": {
  "pause_ms": [10000, 15000]
}
```

说明：
- 这里支持两种写法
- 写成单个数字：`15000`，表示固定停 `15` 秒
- 写成两个数字：`[10000, 15000]`，表示每次都在 `10-15` 秒之间随机停留
- 单位是毫秒
- 正常阅读时，每次上滑后会停这么久
- 翻到下一节后，也会停这么久再开始下一次上滑
- 但“已经到底，准备翻页”这个瞬间不会再额外停 15 秒

常用改法：
- `10000` = 固定停 10 秒
- `[8000, 12000]` = 每次随机停 8-12 秒
- `[10000, 15000]` = 每次随机停 10-15 秒

### 2. 不要 30 次就停

看 [config.json](/E:/haoce/config.json) 里的：

```json
"runtime": {
  "max_page_turns": 0
}
```

说明：
- 这里的“翻页”指的是“成功翻到下一节一次”
- 不是普通的上下滑
- `0` 表示不限次数，会一直跑

如果你以后想限制最多翻多少节，再把它改回正整数就行。

### 3. 调整上滑速度

看 [config.json](/E:/haoce/config.json) 里的：

```json
"scroll": {
  "duration_ms": [450, 650],
  "duration_jitter_ms": 0,
  "settle_ms": 1200
}
```

说明：
- `duration_ms` 也支持两种写法
- 写成单个数字：`520`，表示固定时长
- 写成两个数字：`[450, 650]`，表示每次上滑都在这个区间随机取时长
- `duration_jitter_ms`：在 `duration_ms` 抽出的基础上，再额外加一点随机波动
- `settle_ms`：滑完后等页面稳定再截图的时间

如果你觉得滑动还是太快，优先这样调：
- 先把 `duration_ms` 区间整体调大，比如 `[550, 800]`
- 再把 `pause_ms` 设成一个更长的区间，比如 `[12000, 18000]`

建议：
- 如果你已经把 `duration_ms` 写成区间，通常可以先把 `duration_jitter_ms` 设成 `0`
- 如果你想要更散一点的手感，再给 `duration_jitter_ms` 一个小值

### 4. 选书黑名单文件

看 [config.json](/E:/haoce/config.json) 里的：

```json
"runtime": {
  "selection_blacklist_file": "selection_blacklist.json"
}
```

说明：
- 这个文件会记录“以前已经从书单页选中过的书”
- 它只在“去书单页随机挑新书”时生效
- 不会影响已经在读的书继续刷
- 如果你想重置这个黑名单，直接清空 [selection_blacklist.json](/E:/haoce/selection_blacklist.json) 或删除它就行

## 当前脚本的阅读节奏

现在默认逻辑是：

1. 上滑一次
2. 等页面稳定
3. 用截图差分判断有没有正常移动
4. 如果没到底，就按 `pause_ms` 配置随机停一会儿，再继续上滑
5. 如果连续判断到底，就直接左右翻页，不额外停 15 秒
6. 如果翻页成功，就再随机停一会儿，然后开始新一节的第一次上滑

上下滑和左右翻页都带随机轨迹与随机时长，不会每次都走完全一致的线路。

启动时还有一个额外判断：
- 如果首页 `最近阅读` 第一栏已经显示 `100%`
- 脚本不会再点进这本书
- 会直接进入左上角的 `书香理工` 书单页选下一本

## 一本书什么时候算读完

现在不赌 toast，而是走更稳的流程：

1. 先判断已经到底
2. 尝试翻到下一节
3. 如果翻页没有确认成功，就返回首页
4. 回到首页后先下拉刷新一次，不刷新进度不会同步
5. 再看这本书在 `最近阅读` 里的进度是不是已经 `100%`
6. 如果已经 `100%`，才认为这本书读完了

## 读完后怎么切到下一本

现在切书逻辑是这样的：

1. 回首页
2. 先下拉刷新 `最近阅读`
3. 点左上角的 `书香理工`
4. 进入书单页后，会先点 `分类：...`
5. 从分类弹层里随机选一个非 `重点图书` 的分类
6. 切到新分类后，不会固定选第一本，而是先随机滚到某个书单位置
7. 然后在当前可见候选里随机选一本
8. 如果书单页的 UI 能直接解析出书卡，就随机点解析到的书卡
9. 如果当前页面没有把书卡完整暴露给 UI 树，就退回到“按可见宫格坐标随机顺序尝试”的兜底方案
10. 点开书后：
   - 如果落在“读书报告”页，就点 `继续阅读`
   - 如果落在“书籍详情”页，就点 `开始阅读`
11. 成功进入下一本后，再停 `15` 秒，然后继续正常阅读

启动阶段也会复用同一套“去书单页找下一本”的逻辑。
另外，书单选中过的书会写入 [selection_blacklist.json](/E:/haoce/selection_blacklist.json)，后续随机挑新书时会尽量避开它们，但这不会阻止当前正在刷的书继续阅读。

## 为什么要先下拉刷新

这是当前好策页面的实际行为：
- 回到首页后，阅读进度不会立刻同步
- 必须在这个页面手动下拉一次，`最近阅读` 里的进度才会刷新

所以脚本现在已经把这个动作写进去了。

## 如果脚本正常结束，通常代表什么

常见的正常结束原因有两个：

- 当前书已经读完，而且脚本没有在书单页找到下一本未读完的书
- 你手动把 [config.json](/E:/haoce/config.json) 里的 `max_page_turns` 或 `max_minutes` 设成了正数，并且已经跑到上限

如果日志里看到：

```text
reached max_page_turns=30, stopping
```

那表示你配置了“最多翻 30 次下一节后停止”。

## 直接运行

最简单的方式：

- 双击 [start.bat](/E:/haoce/start.bat)

它会自动做这些事：
- 检查 `python`
- 检查 `adb`
- 安装依赖
- 检查手机、ADB、包名和当前 UI 状态
- 启动脚本

## 常用命令

检查环境：

```powershell
python haoce_reader.py --config config.json doctor
```

开始运行：

```powershell
python haoce_reader.py --config config.json run
```

无限制运行：

```powershell
python haoce_reader.py --config config.json run --max-page-turns 0
```

导出当前 UI：

```powershell
python haoce_reader.py --config config.json dump-ui current.xml
```

截当前屏：

```powershell
python haoce_reader.py --config config.json capture current.png
```

## 文件说明

- [haoce_reader.py](/E:/haoce/haoce_reader.py)
  主脚本，负责 ADB、截图分析、到底判断、翻页和切书
- [config.json](/E:/haoce/config.json)
  配置文件，阅读节奏和运行限制都在这里调
- [start.bat](/E:/haoce/start.bat)
  Windows 下直接双击运行
- [requirements.txt](/E:/haoce/requirements.txt)
  Python 依赖
- [selection_blacklist.json](/E:/haoce/selection_blacklist.json)
  书单选书黑名单，只在“随机选新书”时用于避开以前选过的书

## 环境要求

- Windows
- 已安装 Python
- 已安装 ADB，并且 `adb` 已加到 `PATH`
- 手机已开启 USB 调试
- 电脑已经授权这台手机的 ADB 调试
- 手机里已安装好策，包名为 `app.haoce.com`

安装依赖：

```powershell
pip install -r requirements.txt
```
